### Q1 参考答案
**数据准备**：将原始文档加载并分割成小块，本质是"把大文档变小"
**索引构建**：把文本块转换成向量并存储，本质是"建立可搜索的索引"
**检索优化**：根据用户问题找到最相关的文本块
**生成集成**：将检索结果作为上下文，让 LLM 生成答案

不能跳过或合并的原因：
- 不分割 → 文档太大，无法全部塞进 LLM 的上下文窗口
- 不建索引 → 检索就是大海捞针，无法快速定位相关内容

### Q2 参考答案
嵌入的本质是"语义编码"：把文字转换成多维向量，让语义相似的文字在向量空间中距离更近。

### Q3 参考答案
按 `\n\n` → `\n` → ` ` → `""` 顺序是为了**优先保持语义完整性**：先按段落切，再按行切，再按词切，最后按字切。这样能最大程度保持语义连贯。

### Q4 参考答案
关键词检索找"完全匹配的词"，向量检索找"语义相似的内容"。例如"强化学习"和"增强学习"意思相近但用词不同，关键词检索可能找不到，但向量检索能找到。

### Q5 参考答案
Prompt 模板的作用是：
1. 明确告诉 LLM 任务是什么（根据上下文回答）
2. 指定输入格式（context 和 question 分开）
3. 给出约束条件（必须基于上下文，无法回答时告知）

### Q6 参考答案
LangChain = 高度自定义 + 较陡学习曲线
LlamaIndex = 快速上手 + 封装较多 + 灵活性稍弱
类似 Java 的：MyBatis vs JPA，Spring Boot vs 原生 Spring

### Q7 参考答案
Token 是 LLM 处理文本的基本单位。1 token ≈ 0.75 个英文单词 ≈ 1-2 个中文汉字。
5785 tokens ≈ 4000-6000 中文字符。

### Q8 参考答案
重叠是为了避免上下文被切断导致信息丢失。例如一句话跨两个块，没有重叠时这句话的语义就不完整。
